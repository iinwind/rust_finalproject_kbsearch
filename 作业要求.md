# 作业要求

# 作业目标

本课程期末大作业旨在帮助学生综合运用 Rust 语言知识，完成一个具有实际功能的完整项目。项目应体现：

- Rust 基础语法与类型系统

- 所有权、借用的合理使用

- 模块化设计

- 错误处理

- 工程化开发流程

- 文档与测试能力

---

# 项目形式

单人完成，鼓励根据兴趣自由选题。

**请勿直接抄袭现成开源项目，评审时会做检查。**

---

# 项目选题要求

## 允许自由选题

项目主题不限，但必须满足：

- 主要代码使用 Rust 编写 

- 具有完整、可运行的功能 

- 体现 Rust 的核心特性 

- 项目复杂度达到课程要求 

---

## 不建议的项目类型

以下类型通常难以体现 Rust 特性，不建议选择：

- 纯前端页面项目 

- 简单 CRUD \(增删改查\) 管理系统 

- 仅调用第三方 API 的包装器 

- 主要逻辑由 AI 自动生成的项目 

- Rust 代码占比极低的项目 

## 我的项目选题

- 本地知识库搜索系统 

---

# 项目规模要求

## 单人项目

推荐规模：

- 1500～3000 行有效 Rust 代码 

- 至少 3 个模块 

要求：

- 功能完整 

- 可以独立运行 

- 提供完整文档 

---

# 技术要求

项目必须体现 Rust 工程实践。

### 模块化设计

合理使用：

- mod 

- crate 

- workspace（大型项目推荐） 

---

### 错误处理

要求：

- 使用 Result 

- 合理传播错误 

- 尽量避免大量使用 unwrap/expect 规避错误处理 

---

### Rust 核心特性

项目中应体现：

- ownership / borrowing 

- struct / enum 

- trait 

- 泛型 

- 生命周期（如需要） 

---

### 并发或异步（如需要）

可选：

- thread 

- channel 

- mutex/rwlock 

- async/await 

- tokio 

---

### 测试

至少包含：

- 单元测试 

- 若干关键功能测试 

---

### 工程规范

提交前需通过：

- cargo fmt 

- cargo clippy 

---

### 项目文档（README）

包括使用方法、编译运行方式、依赖说明等

---
